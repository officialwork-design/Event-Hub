# LINE LIFF オフ会管理アプリ 要件定義

## 1. アプリ概要

LINE上からLIFFアプリを起動し、オフ会に関する情報を管理する業務アプリ。

固定構成は以下とする。

LINE
↓
LIFF
↓
GitHub Pages
↓
JavaScript
↓
GAS Web API
↓
Googleスプレッドシート

管理対象は以下。

* オフ会資料
* 準備物
* 当日スケジュール
* 経費
* メモ、注意事項、その他共有事項

まずは全体機能を一気に作らず、**ダッシュボード画面から開発開始**する。

---

## 2. 目的

オフ会運営に必要な情報を、LINE上からすぐ確認・編集できる状態にする。

特に以下を重視する。

* スマホで確認しやすい
* 当日現場で修正できる
* 項目単位ではなく、カテゴリ単位でまとめて編集できる
* Googleスプレッドシートをデータベースとして運用する
* 後から「準備物」「経費」「スケジュール」を追加しやすい構成にする

---

### 4-5. メモ・注意事項画面

例。

* 注意点

## 5. 必要機能

### 5-1. LIFF起動機能

LINEからLIFFを起動する。

必要処理。

* LIFF初期化
* LINEユーザーID取得
* 表示名取得
* 権限確認
* アプリ画面表示

---

### 5-2. ダッシュボード取得機能

GAS Web APIからダッシュボード情報を取得する。

表示内容。

* オフ会基本情報
* 準備物進捗
* 経費合計
* スケジュール概要
* 注意事項
* 最終更新日時

---

### 5-3. ブロック編集機能

重要仕様。

1項目ずつ編集するのではなく、以下のように**ひとくくり単位**で編集する。

例。

準備物ブロック。

```text
カテゴリ：レンタル品
内容：
・射的セット
・輪投げ
・スーパーボールすくい
・ヨーヨー釣り
担当：Andre
状態：確認中
```

このブロック全体をまとめて編集・更新・削除できるようにする。

対象。

* 準備物ブロック
* 経費ブロック
* スケジュールブロック
* メモブロック

---

### 5-4. 更新機能

各ブロックを更新する。

必要処理。

* 入力値チェック
* GAS APIへ送信
* スプレッドシート更新
* 更新後に再取得
* 最終更新日時を更新

---

### 5-5. 削除機能

各ブロックを削除する。

必要処理。

* 削除確認
* GAS APIへ送信
* スプレッドシート上で削除または削除フラグ更新
* 画面再読み込み

推奨は物理削除ではなく、`isDeleted` フラグ管理。

---

### 5-6. カスタム項目追加機能

後から項目を増やせる構成にする。

例。

* チケット
* 景品
* タスク
* 役割分担
* 会場情報
* 連絡先
* URL管理
* 添付資料リンク

## 7. API一覧

GAS Web APIで以下を用意する。

### 7-1. ダッシュボード取得

ファイル名。

`Code.gs`

関数名。

`getDashboard`

用途。

ダッシュボード表示用データを取得する。

リクエスト。

```json
{
  "action": "getDashboard",
  "eventId": "OFFMEETING_001",
  "lineUserId": "LINE_USER_ID"
}
```

レスポンス。

```json
{
  "success": true,
  "dashboard": {},
  "summary": {},
  "blocks": []
}
```

---

### 7-2. ブロック一覧取得

関数名。

`getBlocks`

用途。

準備物、経費、スケジュール、メモを取得する。

リクエスト。

```json
{
  "action": "getBlocks",
  "eventId": "OFFMEETING_001",
  "blockType": "preparation"
}
```

---

### 7-3. ブロック作成

関数名。

`createBlock`

用途。

ブロック単位で新規作成する。

リクエスト。

```json
{
  "action": "createBlock",
  "eventId": "OFFMEETING_001",
  "blockType": "preparation",
  "blockTitle": "レンタル品",
  "blockContent": "射的セット\n輪投げ\nスーパーボールすくい",
  "assignedTo": "Andre"
}
```

---

### 7-4. ブロック更新

関数名。

`updateBlock`

用途。

ブロック単位でまとめて更新する。

リクエスト。

```json
{
  "action": "updateBlock",
  "blockId": "BLOCK_001",
  "blockTitle": "レンタル品",
  "blockContent": "射的セット\n輪投げ\nヨーヨー釣り",
  "amount": 30000,
  "status": "確認中",
  "assignedTo": "Andre"
}
```

---

### 7-5. ブロック削除

関数名。

`deleteBlock`

用途。

ブロックを削除扱いにする。

リクエスト。

```json
{
  "action": "deleteBlock",
  "blockId": "BLOCK_001",
  "lineUserId": "LINE_USER_ID"
}
```

---

### 7-6. 権限確認

関数名。

`checkUserRole`

用途。

LINEユーザーIDから権限を判定する。

リクエスト。

```json
{
  "action": "checkUserRole",
  "lineUserId": "LINE_USER_ID"
}
```

## 開発者への重要指示

このアプリは、最初から全体を作らない。

最初に作るのは **ダッシュボード画面のみ**。

初期作成ファイルは以下。

```text
index.html
style.css
app.js
config.js
Code.gs
```

まずは以下だけ完成させる。

* LIFF起動
* GAS API通信
* スプレッドシートからダッシュボード取得
* ダッシュボード表示
* 準備物、経費、スケジュール、メモの概要表示

準備物・経費・スケジュールの詳細編集機能は、ダッシュボード完成後に追加する。

ただし、データ設計は最初からブロック単位で作ること。

1項目ずつ編集する構成にはしない。
